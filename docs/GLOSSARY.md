# 📖 術語表 (Glossary)

> GigHub 專案的核心術語與定義

---

## 🏗️ 架構術語

| 術語 | 英文 | 定義 |
|------|------|------|
| 基礎層 | Foundation Layer | 處理帳戶、認證、組織等基礎設施 |
| 容器層 | Container Layer | 提供資料隔離、權限控制的邏輯容器 |
| 業務層 | Business Layer | 實現具體業務功能（任務、日誌等）|
| 藍圖 | Blueprint | 邏輯容器，提供資料隔離和上下文共享 |

---

## 👥 角色術語

| 術語 | 英文 | 定義 |
|------|------|------|
| 專案經理 | Project Manager | 藍圖最高權限者，可管理所有設定 |
| 工地主任 | Site Director | 管理施工進度、審核日誌 |
| 施工人員 | Worker | 執行任務、上傳現場照片 |
| 品管人員 | QA Staff | 負責品質驗收、填寫檢查清單 |
| 觀察者 | Observer | 只讀權限，查看進度報告 |

---

## 📋 業務術語

| 術語 | 英文 | 定義 |
|------|------|------|
| 任務 | Task | 工作項目，支援樹狀結構與狀態追蹤 |
| 施工日誌 | Diary | 每日施工紀錄，包含照片與工時 |
| 品質驗收 | Quality Acceptance | 工程品質檢查與簽核流程 |
| 待辦事項 | Todo | 個人待辦清單 |

---

## 🔧 技術術語

| 術語 | 英文 | 定義 |
|------|------|------|
| RLS | Row Level Security | PostgreSQL 資料列級安全控制 |
| Signals | Angular Signals | Angular 響應式狀態管理機制 |
| Repository | Repository Pattern | 資料存取抽象層模式 |
| Store | State Store | 狀態管理容器 |
| MCP | Model Context Protocol | AI 模型上下文協議 |

---

## 🤖 AI 術語

| 術語 | 英文 | 定義 |
|------|------|------|
| Agent | AI Agent | 專門化的 AI 代理，處理特定領域任務 |
| Prompt | Prompt Template | 可重複使用的 AI 提示模板 |
| Instruction | Coding Instruction | 自動套用的編碼標準指引 |
| Blueprint | Code Blueprint | 標準化的程式碼生成模板 |
